🔌Section 1: Developer Access

SSH into Server via Visual Studio Code

Step-by-Step 1. Install the Remote - SSH extension in VS Code.
2. Press F1 → Remote-SSH: Connect to Host...
3. Type: arjun@192.168.1.16
4. Enter password and open folder: /mnt/ssd-fast/FitnessClassBookingSystem

Managing the Linux GUI (Display Manager)

To maximize performance during heavy builds, you may wish to stop the Linux Desktop environment.

Bash - System Control
# To STOP the GUI (Saves ~1GB RAM)
sudo systemctl stop gdm3

# To START the GUI
sudo systemctl start gdm3

Samba Setup (The Z: Drive Bridge)

View smb.conf & Windows Mapping
Linux Config: Edit /etc/samba/smb.conf and add: 
[FitnessBooking]
   path = /mnt/ssd-fast/FitnessClassBookingSystem
   valid users = arjun
   read only = no
   create mask = 0775

Windows: Right-click "This PC" → "Map Network Drive" → Drive: Z: → Folder: \\192.168.1.16\FitnessBooking

🗄️Section 2: SQL Server Database

Deployment of MS SQL Server 2022 with NodePort for external SSMS connectivity.

SQL Deployment YAML (Auto-Collapsed) 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: sql-server-deploy
spec:
  replicas: 1
  selector:
    matchLabels:
      app: sql-server
  template:
    metadata:
      labels:
        app: sql-server
    spec:
      containers:
      - name: mssql
        image: mcr.microsoft.com/mssql/server:2022-latest
        env:
        - name: ACCEPT_EULA
          value: "Y"
        - name: MSSQL_SA_PASSWORD
          value: "YourPassword123!"
---
apiVersion: v1
kind: Service
metadata:
  name: sql-server-service
spec:
  type: NodePort
  ports:
    - port: 1433
      nodePort: 31433
  selector:
    app: sql-server

Common SQL CLI Operations

Kubectl Commands
# Connect to SQL shell inside the pod
kubectl exec -it $(kubectl get pod -l app=sql-server -o name) -- /opt/mssql-tools/bin/sqlcmd -S localhost -U sa -P YourPassword123!

# Create the database manually if migrations aren't used
CREATE DATABASE AccountDb;
GO

📊Section 3: Full-Stack Tracing

Jaeger 1.60 (Pinned Version)

Why v1.60? Newer "latest" builds of Jaeger have a CSS bug causing white text on white backgrounds. Version 1.60 is the stable choice for ArjunServer.
Jaeger 1.60 Deployment YAML 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: jaeger
spec:
  template:
    spec:
      containers:
      - name: jaeger
        image: jaegertracing/all-in-one:1.60
        env:
        - name: COLLECTOR_OTLP_ENABLED
          value: "true"

OpenTelemetry Collector

Collector Pipeline Config 
receivers:
  otlp:
    protocols:
      grpc:
        endpoint: 0.0.0.0:4317
exporters:
  otlp:
    endpoint: "jaeger:4317"
    tls:
      insecure: true
service:
  pipelines:
    traces:
      receivers: [otlp]
      exporters: [otlp]

🚀Section 4: .NET 9 Account API

API Deployment YAML (NodePort: 30005) 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: account-api-deploy
spec:
  template:
    spec:
      containers:
      - name: account-api
        image: account-api:v3
        env:
        - name: ConnectionStrings__DefaultConnection
          value: "Server=sql-server-service;Database=AccountDb;User Id=sa;Password=YourPassword123!;TrustServerCertificate=True"
        - name: OTEL_SERVICE_NAME
          value: "AccountAPI"

Program.cs Telemetry Implementation

C# Code (Resource & Exemplars) 
builder.Services.AddOpenTelemetry()
    .ConfigureResource(r => r.AddService("AccountAPI"))
    .WithTracing(t => t
        .AddAspNetCoreInstrumentation()
        .AddSqlClientInstrumentation(opt => opt.SetDbStatementForText = true)
        .AddOtlpExporter(o => o.Endpoint = new Uri("http://otel-collector:4317")))
    .WithMetrics(m => m
        .AddRuntimeInstrumentation()
        .SetExemplarFilter(ExemplarFilterType.TraceBased)
        .AddOtlpExporter());

🚀Section 4: .NET 9 Account API

#!/bin/bash echo "--- 🚀 Starting Deployment of AccountAPI ---" # 1. Pull the latest code from GitHub echo "📥 Pulling latest changes..." git pull origin main # 2. 💻Section 5: The CLI Masterlist

Essential commands for moving through the project directories and managing the cluster.

Navigation from arjun@arjunserver:~
# Go to Project Root
cd /mnt/ssd-fast/FitnessClassBookingSystem

# Go to Accounts API Source
cd /mnt/ssd-fast/FitnessClassBookingSystem/src/Accounts/Presentation

# Go to Deployment YAMLs
cd /mnt/ssd-fast/FitnessClassBookingSystem/src/Accounts
Kubernetes Status & Troubleshooting
# Check all Pods status
kubectl get pods

# Live Tail Logs for the API
kubectl logs -f deployment/account-api-deploy

# Check Deployment History
kubectl rollout history deployment/account-api-deploy

# Complete System Refresh (Apply all YAMLs)
kubectl apply -f jaeger.yaml
kubectl apply -f otel-config.yaml
kubectl apply -f account-deploy.yaml
Critical Failure Point "I built the image with nerdctl, but Kubectl says ImagePullBackOff!" ✅ Solution You must include the Kubernetes namespace during build:
sudo nerdctl -n k8s.io build -t account-api:v3 .
